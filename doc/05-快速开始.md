## 后端启动

### 5.4 克隆项目

```bash
git clone <repository-url>
cd iboot/iboot-admin
```

### 5.5 配置数据库

创建 MySQL 数据库：
```sql
CREATE DATABASE iboot DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'iboot'@'localhost' IDENTIFIED BY 'iboot123';
GRANT ALL PRIVILEGES ON iboot.* TO 'iboot'@'localhost';
FLUSH PRIVILEGES;
```

### 5.6 配置文件

**开发环境配置（application-dev.yml）：**

修改 `iboot-admin/src/main/resources/application-dev.yml`：

```yaml
spring:
  # 数据源配置
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      driver-class-name: com.mysql.cj.jdbc.Driver
      url: jdbc:mysql://localhost:3306/iboot?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
      username: iboot
      password: iboot123
      # 连接池配置
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000

  # Redis 配置
  data:
    redis:
      host: localhost
      port: 6379
      password:
      database: 0
      timeout: 10s
      lettuce:
        pool:
          min-idle: 0
          max-idle: 8
          max-active: 8

  # Flyway 配置
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true
    validate-on-migrate: false
    out-of-order: true

# 服务器配置
server:
  port: 8080

# JWT 配置
jwt:
  secret: dev-secret-key-for-jwt-token-generation-minimum-256-bits
  expiration: 604800        # 7 天（秒）
  session-timeout: 30       # 30 分钟滑动过期

# 日志配置
logging:
  level:
    root: INFO
    com.iboot.admin: DEBUG
    org.mybatis: DEBUG
  file:
    name: logs/iboot-admin.log
```

**生产环境配置（使用环境变量）：**

```yaml
spring:
  datasource:
    druid:
      url: jdbc:mysql://${DB_HOST}:${DB_PORT:3306}/${DB_NAME:iboot}
      username: ${DB_USERNAME}
      password: ${DB_PASSWORD}
  data:
    redis:
      host: ${REDIS_HOST}
      password: ${REDIS_PASSWORD}

jwt:
  secret: ${JWT_SECRET}
```

### 5.7 数据库迁移

Flyway 会自动执行数据库迁移脚本，首次启动时会自动创建表结构。

**迁移脚本位置：**
```
iboot-admin/src/main/resources/db/migration/
├── V1__init_schema.sql          # 初始化表结构
├── V2__add_sys_user.sql         # 添加用户表
├── V3__add_sys_role.sql         # 添加角色表
├── V4__add_sys_menu.sql         # 添加菜单表
├── V5__add_sys_dept.sql         # 添加部门表
├── V6__add_sys_dict.sql         # 添加字典表
├── V7__add_sys_config.sql       # 添加配置表
├── V8__add_sys_job.sql          # 添加定时任务表
├── V9__add_sys_file.sql         # 添加文件表
├── V10__add_sys_log.sql         # 添加日志表
└── R__insert_dict_data.sql      # 可重复执行，插入字典数据
```

**手动执行迁移脚本：**
```bash
# 方式一：使用 MySQL 命令行
mysql -u iboot -p iboot < iboot-admin/src/main/resources/db/migration/V1__init_schema.sql

# 方式二：使用 Flyway 命令行
flyway -url="jdbc:mysql://localhost:3306/iboot" -user="iboot" -password="iboot123" migrate
```

### 5.8 启动后端

**方式一：使用 Maven 启动（推荐开发使用）**
```bash
cd iboot-admin
mvn clean install -DskipTests
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

**方式二：运行 JAR 包**
```bash
cd iboot-admin
mvn clean package -DskipTests
java -jar target/iboot-admin.jar --spring.profiles.active=dev
```

**方式三：使用 IDEA 启动**
1. 找到 `IbootAdminApplication.java`
2. 右键 -> Run 'IbootAdminApplication'

### 5.12 配置文件

**开发环境配置（.env）：**

修改 `iboot-portal/.env` 文件：

```bash
# 应用标题
VITE_APP_TITLE=iBoot 管理系统

# API 基础路径
VITE_API_BASE_URL=/iboot/api

# 是否启用 sourcemap
VITE_SOURCE_MAP=true

# 版本号
VITE_APP_VERSION=1.0.0
```

**代理配置（vite.config.ts）：**

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  server: {
    port: 3000,
    host: true,
    proxy: {
      '/iboot/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/iboot\/api/, '/api')
      }
    }
  },
  css: {
    preprocessorOptions: {
      scss: {
        additionalData: `@use "@/styles/variables.scss" as *;`
      }
    }
  },
  build: {
    outDir: 'dist',
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true
      }
    },
    rollupOptions: {
      output: {
        manualChunks: {
          'vue-vendor': ['vue', 'vue-router', 'pinia'],
          'element-plus': ['element-plus'],
          'echarts': ['echarts']
        }
      }
    }
  }
})
```

```bash
cd docker
docker-compose up -d
```

启动的服务包括：
| 服务 | 端口 | 描述 |
|------|------|------|
| MySQL | 3306 | 数据库 |
| Redis | 6379 | 缓存 |
| MinIO | 9000/9001 | 对象存储 |
| Prometheus | 9090 | 指标监控 |
| Grafana | 3000 | 可视化仪表板 |
| Elasticsearch | 9200/9300 | 搜索引擎 |

| Kibana | 5601 | 日志可视化 |

```bash
docker-compose ps
```

应显示所有服务状态为 `Up`。

### 5.22 查看日志

```bash

docker-compose logs -f redis
```


## 常见问题

### 5.27 后端启动失败

**问题 1：端口被占用**
```
Port 8080 was already in use.
```

**解决方案：**
```bash
# 查找占用端口的进程
lsof -i :8080
# 杀死进程
kill -9 <PID>

# 或修改配置更换端口
server:
  port: 8081
```

**问题 2：数据库连接失败**
```
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```

**解决方案：**
1. 检查 MySQL 是否启动：`docker ps | grep mysql` 或 `systemctl status mysqld`
2. 检查数据库连接配置是否正确
3. 检查防火墙设置

**问题 3：Redis 连接失败**
```
io.lettuce.core.RedisConnectionException: Unable to connect to localhost:6379
```

**解决方案：**
1. 检查 Redis 是否启动：`docker ps | grep redis` 或 `systemctl status redis`
2. 检查 Redis 配置：`redis-cli ping` 应返回 `PONG`

### 5.28 前端启动失败

**问题 1：依赖安装失败**
```
npm ERR! code ERESOLVE
```

**解决方案：**
```bash
# 清理缓存
npm cache clean --force
# 删除 node_modules
rm -rf node_modules package-lock.json
# 重新安装
npm install
```

**问题 2：开发服务器启动失败**
```
Error: listen EADDRINUSE: address already in use 0.0.0.0:3000
```

**解决方案：**
```bash
# 查找占用端口的进程
lsof -i :3000
# 杀死进程
kill -9 <PID>

# 或修改 vite.config.ts 更换端口
server: {
  port: 3001
}
```

**问题 3：代理请求失败**
```
Error: connect ECONNREFUSED 127.0.0.1:8080
```

**解决方案：**
1. 确认后端服务已启动
2. 检查代理配置 `vite.config.ts` 中的 `target` 地址
3. 检查后端服务监听的地址和端口

### 5.29 Docker 相关问题

**问题 1：容器启动失败**
```
Error starting userland proxy: listen tcp4 0.0.0.0:3306: bind: address already in use
```

**解决方案：**
```bash
# 停止本地 MySQL 服务
sudo systemctl stop mysqld

# 或修改 docker-compose.yml 更换端口
ports:
  - "3307:3306"
```

**问题 2：容器间网络不通**
```
Unable to connect to mysql:3306
```

**解决方案：**
1. 确保所有服务在同一个 docker-compose 网络中
2. 使用服务名而非 localhost 访问（如 `mysql` 而非 `127.0.0.1`）
3. 检查网络配置：`docker network ls`

### 5.30 登录问题

**问题：无法登录**
```
登录失败：用户名或密码错误
```

**解决方案：**
1. 确认数据库已正确初始化
2. 检查默认账号是否存在：
   ```sql
   SELECT * FROM sys_user WHERE username = 'admin';
   ```
3. 检查密码加密是否正确（BCrypt）
4. 查看后端日志定位具体错误

**问题：登录后提示无权限**
```
403 Forbidden
```

**解决方案：**
1. 检查用户是否分配了角色
2. 检查角色是否分配了菜单权限
3. 清除浏览器缓存后重新登录
