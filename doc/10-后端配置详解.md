## 配置文件概述

### 10.1 配置文件位置

```
iboot-admin/src/main/resources/
├── application.yml              # 主配置文件
├── application-dev.yml          # 开发环境配置
├── application-test.yml         # 测试环境配置
├── application-prod.yml         # 生产环境配置
├── banner.txt                   # 启动 Banner
└── db/migration/                # Flyway 迁移脚本
```

### 10.2 配置文件加载顺序

Spring Boot 按照以下顺序加载配置文件（后加载的覆盖先加载的）：

1. `application.yml` (基础配置)
2. `application-{profile}.yml` (环境特定配置)
3. 外部配置文件 (通过 `spring.config.location` 指定)
4. 环境变量
5. JVM 系统属性
6. 命令行参数

## 服务器配置

### 10.3 Server 配置

```yaml
server:
  port: 8080                          # 服务端口
  servlet:
    context-path: /                   # 上下文路径
    encoding:
      charset: UTF-8                  # 编码格式
      enabled: true                   # 启用编码过滤
      force: true                     # 强制设置编码
  compression:
    enabled: true                     # 启用 Gzip 压缩
    min-response-size: 1024           # 最小压缩大小 (字节)
    mime-types: text/html,text/xml,text/plain,application/json,application/javascript
  tomcat:
    uri-encoding: UTF-8               # URI 编码
    max-threads: 200                  # 最大线程数
    min-spare-threads: 10             # 最小空闲线程数
    connection-timeout: 20000         # 连接超时时间 (毫秒)
    keep-alive-timeout: 30000         # 保持连接超时时间 (毫秒)
  shutdown: graceful                  # 优雅关闭
```

**配置说明：**

| 配置项 | 默认值 | 描述 |
|--------|--------|------|
| port | 8080 | HTTP 服务端口 |
| servlet.context-path | / | 应用上下文路径 |
| servlet.encoding.charset | UTF-8 | 请求响应编码 |
| compression.enabled | false | 是否启用 Gzip 压缩 |
| compression.min-response-size | 2048 | 最小压缩响应大小 |
| tomcat.max-threads | 200 | Tomcat 最大工作线程数 |
| tomcat.min-spare-threads | 10 | Tomcat 最小空闲线程数 |
| shutdown | immediate | 关闭模式 (immediate/graceful) |

## 数据源配置

### 10.6 Druid 连接池配置

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      # 基本连接配置
      driver-class-name: com.mysql.cj.jdbc.Driver
      url: jdbc:mysql://localhost:3306/iboot_admin?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
      username: root
      password: root

      # 连接池基本配置
      initial-size: 5                 # 初始连接数
      min-idle: 5                     # 最小空闲连接数
      max-active: 20                  # 最大活跃连接数
      max-wait: 60000                 # 获取连接最大等待时间 (毫秒)

      # 连接检测配置
      time-between-eviction-runs-millis: 60000      # 检测间隔
      min-evictable-idle-time-millis: 300000        # 最小空闲时间
      validation-query: SELECT 1                    # 验证查询
      test-while-idle: true           # 空闲时检测
      test-on-borrow: true            # 借出时检测
      test-on-return: false           # 归还时检测

      # 预编译语句配置
      pool-prepared-statements: true
      max-pool-prepared-statement-per-connection-size: 20

      # 监控配置
      filters: stat,wall,slf4j
      filter:
        slf4j:
          enabled: true
          statement-create-after-log-enabled: true
          statement-close-after-log-enabled: true
          result-set-open-after-log-enabled: true
          result-set-close-after-log-enabled: true
      connection-properties: druid.stat.mergeSql=true;druid.stat.slowSqlMillis=5000

      # Web 监控配置
      web-stat-filter:
        enabled: true
        url-pattern: /*
        exclusions: '*.js,*.gif,*.jpg,*.png,*.css,*.ico,/druid/*'

      # 监控页面配置
      stat-view-servlet:
        enabled: true
        url-pattern: /druid/*
        reset-enable: false           # 禁用重置功能
        # allow: 127.0.0.1            # 允许访问的 IP
        # deny:                       # 拒绝访问的 IP
```

**配置说明：**

| 配置项 | 推荐值 | 描述 |
|--------|--------|------|
| initial-size | 5 | 启动时创建的连接数 |
| min-idle | 5 | 最小空闲连接数 |
| max-active | 20 | 最大连接数，根据并发量调整 |
| max-wait | 60000 | 获取连接超时时间 |
| validation-query | SELECT 1 | 验证连接有效的 SQL |
| test-while-idle | true | 空闲时检测连接有效性 |
| filters | stat,wall,slf4j | 监控统计、防火墙、日志 |

### 10.7 连接池优化建议

**开发环境：**
```yaml
initial-size: 2

```

**生产环境：**

## Redis 配置

### 10.8 Redis 连接配置

详细的二级缓存配置请参考 [12-二级缓存设计](./12-二级缓存设计.md)。

```yaml
spring:
  data:
    redis:
      host: localhost                 # Redis 主机
      port: 6379                      # Redis 端口
      password:                       # 密码 (无密码留空)
      database: 0                     # 数据库索引
      timeout: 10s                    # 连接超时

      # Lettuce 连接池配置
      lettuce:
        pool:
          min-idle: 0                 # 最小空闲连接
          max-idle: 8                 # 最大空闲连接
          max-active: 8               # 最大连接数
          max-wait: -1ms              # 获取连接最大等待时间 (-1 表示不等待)
```

**配置说明：**

| 配置项 | 默认值 | 描述 |
|--------|--------|------|
| host | localhost | Redis 服务器地址 |
| port | 6379 | Redis 端口 |
| database | 0 | 使用的数据库索引 (0-15) |
| timeout | 10s | 连接超时时间 |
| lettuce.pool.max-active | 8 | 最大连接数 |
| lettuce.pool.max-idle | 8 | 最大空闲连接数 |
| lettuce.pool.min-idle | 0 | 最小空闲连接数 |

- **[12-二级缓存设计](./12-二级缓存设计.md)** - 包含完整的 Caffeine + Redis 二级缓存架构设计、缓存策略、分布式一致性方案

## Flyway 配置

### 10.11 Flyway 数据库迁移配置

```yaml
spring:
  flyway:
    enabled: true                     # 启用 Flyway
    locations: classpath:db/migration # 迁移脚本位置
    baseline-on-migrate: true         # 非空库自动创建基线
    validate-on-migrate: false        # 迁移前不验证
    baseline-version: 0               # 基线版本
    out-of-order: true                # 允许乱序执行
    ignore-migration-patterns: *:missing,*:future,*:ignored
```

**配置说明：**

| 配置项 | 默认值 | 描述 |
|--------|--------|------|
| enabled | true | 是否启用 Flyway |
| locations | classpath:db/migration | 迁移脚本位置 |
| baseline-on-migrate | false | 自动创建基线 |
| validate-on-migrate | true | 迁移前验证 |
| out-of-order | false | 允许乱序执行 |

**迁移脚本命名规范：**

```
db/migration/
├── V1__init_schema.sql              # V=版本，初始化
├── V2__add_user_table.sql           # 添加用户表
├── V3__add_role_table.sql           # 添加角色表
├── V1.1__add_user_columns.sql       # 小版本，添加字段
└── R__insert_dict_data.sql          # R=可重复执行
```

## Quartz 定时任务配置

### 10.12 Quartz 配置

```yaml
spring:
  quartz:
    job-store-type: jdbc              # 使用 JDBC 存储
    jdbc:
      initialize-schema: never        # 不自动初始化表
    scheduler-name: iBootScheduler    # 调度器名称
    overwrite-existing-jobs: true     # 覆盖已存在的任务
    wait-for-jobs-to-complete-on-shutdown: true  # 关闭时等待任务完成
    properties:
      org.quartz.jobStore.tablePrefix: QRTZ_           # 表前缀
      org.quartz.jobStore.isClustered: false           # 是否集群
      org.quartz.jobStore.misfireThreshold: 60000      # 错失触发阈值
      org.quartz.threadPool.threadCount: 10            # 线程数
      org.quartz.threadPool.threadPriority: 5          # 线程优先级
```

**配置说明：**

| 配置项 | 默认值 | 描述 |
|--------|--------|------|
| job-store-type | memory | 存储类型 (memory/jdbc/jdbc-cmt) |
| jdbc.initialize-schema | embedded | 自动初始化 Schema |
| scheduler-name | scheduler | 调度器名称 |
| overwrite-existing-jobs | false | 覆盖已有任务 |
| properties.org.quartz.threadPool.threadCount | 10 | 工作线程数 |
| properties.org.quartz.jobStore.isClustered | false | 是否集群模式 |

## JWT 认证配置

### 10.13 JWT 配置

```yaml
jwt:
  secret: iboot-admin-secret-key-for-jwt-token-generation-minimum-256-bits
  expiration: 604800      # JWT 签名兜底过期（秒），7 天
  session-timeout: 30     # Redis 滑动过期（分钟），实际控制会话时长
```

**配置说明：**

| 配置项 | 描述 |
|--------|------|
| secret | JWT 签名密钥，至少 256 位（32 字符） |
| expiration | JWT Token 最大有效期（秒） |
| session-timeout | Redis 中会话的滑动过期时间（分钟） |

**生产环境建议：**
```yaml
jwt:
  secret: ${JWT_SECRET:}  # 从环境变量读取
  expiration: 604800
  session-timeout: 30
```

## Actuator 监控配置

### 10.14 Actuator 端点配置

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,beans,env,configprops,mappings,logfile,threaddump,caches,scheduledtasks,conditions,prometheus
      base-path: /actuator        # Actuator 基础路径
  endpoint:
    health:
      show-details: always        # 显示健康详情
      roles: ADMIN                # 访问角色
    logfile:
      external-file: logs/iboot-admin.log  # 外部日志文件
  info:
    git:
      mode: full                  # Git 信息模式
    build:
      enabled: true               # 启用构建信息
    env:
      enabled: true               # 启用环境变量
  metrics:
    distribution:
      percentiles-histogram:
        http.server.requests: true
        jdbc.connections.usage: true
        cache.requests: true
      percentiles:
        http.server.requests: 0.5, 0.95, 0.99  # 响应时间百分位
    enable:
      jvm: true
      system: true
      logback: true
      tomcat: true
      spring-data: true
    tags:
      application: iboot-admin    # 应用标签
      namespace: default          # 命名空间标签
```

**可用端点说明：**

| 端点 | 描述 |

| metrics | 性能指标 |
| beans | Spring Beans 列表 |
| env | 环境变量 |
| configprops | 配置属性 |
| mappings | 请求映射 |
| logfile | 日志文件 |
| threaddump | 线程转储 |
| caches | 缓存信息 |
