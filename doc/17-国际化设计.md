# 17-国际化设计

## 1. 概述

iBoot 项目支持完整的多语言国际化（i18n）能力，包括简体中文（zh-CN）和英文（en-US）。系统采用前后端分离的国际化方案：

- **前端**：使用 vue-i18n v9+ 进行多语言管理
- **后端**：使用 Spring MessageSource 进行消息国际化

## 2. 前端国际化架构

### 2.1 vue-i18n 配置

前端使用 vue-i18n v9+ 的 Composition API 进行国际化配置。

**配置文件位置**: `iboot-portal/src/locales/index.ts`

**核心配置项**:
- `legacy: false` - 禁用遗留 API，使用 Composition API
- `locale` - 从 localStorage 读取用户语言偏好，默认为 `zh-CN`
- `fallbackLocale` - 回退语言设置为 `zh-CN`
- `messages` - 模块化加载语言包

### 2.3 语言包加载机制

1. **初始化加载**: 应用启动时加载默认语言的所有模块
2. **懒加载**: 路由级别的模块可按需加载（可选优化）
3. **缓存机制**: 已加载的语言包缓存在内存中，避免重复请求

### 2.5 语言切换组件

系统提供 `LocalePicker` 组件用于语言切换：

```vue
<template>

      <el-dropdown-menu>
        <el-dropdown-item

          :key="item.value"
          :command="item.value"

          {{ item.label }}
        </el-dropdown-item>
      </el-dropdown-menu>
    </template>
  </el-dropdown>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'
import { useLocale } from '@/composables/useLocale'

const { t } = useI18n()
const { currentLocale, changeLocale } = useLocale()

const locales = computed(() => [
  { label: t('common.locale.zh-CN'), value: 'zh-CN' },
  { label: t('common.locale.en-US'), value: 'en-US' },
])

const handleLocaleChange = (locale: string) => {
  changeLocale(locale)
}
</script>
```

## 3. 后端国际化架构

### 3.1 MessageSource 配置

后端使用 Spring 的 `ResourceBundleMessageSource` 进行消息国际化。

**配置类**: `com.iboot.admin.infrastructure.config.I18nConfig`

```java
@Configuration
public class I18nConfig {

    @Bean
    public MessageSource messageSource() {
        ResourceBundleMessageSource source = new ResourceBundleMessageSource();
        source.setBasenames(
            "i18n/messages",           // 通用消息
            "i18n/exception/exceptions", // 异常消息
            "i18n/business/business"    // 业务消息
        );
        source.setDefaultEncoding("UTF-8");
        source.setCacheSeconds(3600);
        return source;
    }

    @Bean
    public LocaleResolver localeResolver() {
        AcceptHeaderLocaleResolver resolver = new AcceptHeaderLocaleResolver();
        resolver.setDefaultLocale(Locale.SIMPLIFIED_CHINESE);
        return resolver;
    }
}
```

### 3.4 I18nUtil 工具类

提供便捷的消息获取方法：

```java
@Component
public class I18nUtil {

    @Autowired
    private MessageSource messageSource;

    /**
     * 获取当前语言的消息
     */
    public String getMessage(String code) {
        Locale locale = LocaleContextHolder.getLocale();
        return getMessage(code, locale);
    }

    /**
     * 获取带参数的消息
     */
    public String getMessage(String code, Object... args) {
        Locale locale = LocaleContextHolder.getLocale();
        try {
            return messageSource.getMessage(code, args, locale);
        } catch (NoSuchMessageException e) {
            return code; // 回退到消息码本身
        }
    }
}
```

### 3.5 LocaleResolver 工作原理

1. **请求到达**: Spring Security 过滤器链处理请求
2. **解析语言**: `LocaleResolver` 从 `Accept-Language` 头解析语言
3. **设置上下文**: 将解析的 Locale 存入 `LocaleContextHolder`
4. **消息查找**: 业务代码通过 `I18nUtil.getMessage()` 获取对应语言消息
5. **默认回退**: 未指定语言时默认使用简体中文（zh-CN）

## 4. 语言切换流程

### 4.1 前后端交互流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端 UI    │     │  后端 API    │     │  持久化存储  │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       │ 1. 用户选择语言    │                   │
       ├──────────────────►│                   │
       │                   │                   │
       │                   │ 2. 设置 Cookie    │
       │                   ├──────────────────►│
       │                   │                   │
       │ 3. 返回成功响应   │                   │
       │◄──────────────────┤                   │
       │                   │                   │
       │ 4. 更新 localStorage                 │
       ├──────────────────────────────────────►│
       │                   │                   │
       │ 5. 更新 vue-i18n locale              │
       │ 6. 更新 Element Plus locale          │
       │ 7. 页面重新渲染                      │
       ▼                   ▼                   ▼
```

### 4.2 API 接口

**切换语言**:
```http
POST /api/i18n/locale?locale=en-US
```

**响应示例**:
```json
{
  "code": 200,
  "message": "Language changed successfully",
  "data": {
    "locale": "en-US"
  }
}
```

**获取当前语言**:
```http
GET /api/i18n/locale
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": "zh-CN"
}
```

### 4.3 持久化机制

| 存储位置 | 用途 | 过期时间 |
|---------|------|---------|
| localStorage | 前端保存用户语言偏好 | 永久 |
| Cookie | 后端识别用户语言偏好 | 1 年 |
| document.documentElement.lang | 当前页面语言状态 | 会话级 |

## 5. 消息码响应格式

### 5.1 Result 类国际化

后端 `Result` 类支持消息码方式的响应：

```java
// 成功响应（带消息码）
public static <T> Result<T> success(String code, T data) {
    Result<T> result = new Result<>();
    result.setCode(200);
    result.setData(data);
    result.setMessage(I18nUtil.getMessage(code));
    return result;
}

// 错误响应（带消息码）
public static Result<Void> error(String code) {
    Result<Void> result = new Result<>();
    result.setCode(500);
    result.setMessage(I18nUtil.getMessage(code));
    return result;
}
```

### 5.2 全局异常处理

`GlobalExceptionHandler` 自动处理异常消息的国际化：

```java
@ExceptionHandler(BusinessException.class)
public Result<Void> handleBusinessException(BusinessException e) {
    String message = I18nUtil.getMessage(e.getMessageCode());
    return Result.error(message);
}
```

## 6. 最佳实践

### 6.1 翻译键命名

- 使用 `模块。功能。键` 的层级结构
- 键名使用小写字母和下划线
- 避免在键名中使用中文或特殊字符

**好的示例**:
```
login.title
login.username_placeholder
system.user.add_success
```

**坏的示例**:
```
登录标题
userAddSuccess
temp1
```

### 6.2 参数化翻译

使用占位符避免字符串拼接：

```json
// 推荐
{
  "welcome": "欢迎，{0}！",
  "item_count": "共 {total} 项"
}
```

```typescript
// 使用
t('welcome', username)
t('item_count', { total: 100 })
```

### 6.3 避免硬编码

**推荐**:
```vue
<el-button>{{ t('common.submit') }}</el-button>
```

**不推荐**:
```vue
<el-button>提交</el-button>
```

### 6.4 动态语言切换

在需要动态获取当前语言的场景（如时间格式化）：

```typescript
const formatTime = (timeString: string) => {
  const locale = document.documentElement.lang || 'zh-CN'
  return new Date(timeString).toLocaleDateString(locale, {
    month: 'short',
    day: 'numeric',
    hour: '2-digit',
    minute: '2-digit'
  })
}
```

## 7. 常见问题

### Q: 添加新语言需要什么步骤？

1. 在 `iboot-portal/src/locales/` 下创建新语言目录（如 `ja-JP/`）
2. 复制现有语言包文件结构
3. 翻译所有 JSON 文件内容
4. 在 `index.ts` 中注册新语言
5. 后端添加对应的 `.properties` 文件
6. 测试语言切换功能

### Q: 如何确保翻译的完整性？

- 使用脚本对比不同语言包的键数量
- 在 CI/CD 中加入翻译完整性检查
- 定期审查未翻译的键（回退到消息码本身的情况）

### Q: 语言切换后部分文本未更新？

检查以下情况：
1. 是否使用了 `t()` 函数而非硬编码
2. 组件是否响应式地监听 locale 变化
3. Element Plus locale 是否同步更新
4. 动态渲染的内容是否使用了正确的语言包
