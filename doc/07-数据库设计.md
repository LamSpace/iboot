## 数据库概述

### 7.1 数据库配置

| 配置项 | 值 |
|--------|------|
| 数据库类型 | MySQL 8.x |
| 字符集 | utf8mb4 |
| 排序规则 | utf8mb4_unicode_ci |
| 存储引擎 | InnoDB |

### 7.2 数据库命名规范

- 数据库名：`iboot`
- 表名：`sys_模块 _ 实体` 格式，使用小写字母和下划线
- 字段名：小写字母和下划线
- 主键：统一使用 `id`
- 索引：`idx_字段名`
- 唯一索引：`uk_字段名`

### 7.3 公共字段

所有业务表包含以下公共字段：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| create_by | varchar(64) | 创建人 |
| create_time | datetime | 创建时间 |
| update_by | varchar(64) | 更新人 |
| update_time | datetime | 更新时间 |
| deleted | tinyint(1) | 逻辑删除标识（0-未删除，1-已删除） |

## Flyway 数据库迁移

### 7.4 Flyway 配置

**application.yml 配置：**
```yaml
spring:
  flyway:
    enabled: true                     # 启用 Flyway
    locations: classpath:db/migration # 迁移脚本位置
    baseline-on-migrate: true         # 非空库自动创建基线
    validate-on-migrate: false        # 迁移前不验证
    baseline-version: 0               # 基线版本
    out-of-order: true                # 允许乱序执行
    ignore-migration-patterns: *:missing,*:future,*:ignored
```

### 7.5 迁移脚本命名规范

| 前缀 | 描述 | 示例 |
|------|------|------|
| V | 版本迁移（Versioned） | V1__init_schema.sql |
| R | 可重复迁移（Repeatable） | R__insert_dict_data.sql |
| U | 撤销迁移（Undo） | U1__add_table.sql |

**版本号规则：**
- 主版本号。次版本号：V1.0、V1.1、V2.0
- 小版本升级可以使用小数点：V1.1、V1.2
- 紧急修复可以使用三位版本号：V1.1.1

### 7.7 V1__init_schema.sql 示例

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS iboot DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE iboot;

-- 用户表
CREATE TABLE IF NOT EXISTS sys_user (
    id bigint(20) NOT NULL AUTO_INCREMENT COMMENT '用户 ID',
    username varchar(64) NOT NULL COMMENT '用户名',
    password varchar(255) NOT NULL COMMENT '密码 (BCrypt 加密)',
    nickname varchar(64) DEFAULT NULL COMMENT '昵称',
    email varchar(128) DEFAULT NULL COMMENT '邮箱',
    phone varchar(20) DEFAULT NULL COMMENT '手机号',
    gender tinyint(1) DEFAULT 0 COMMENT '性别 (0-未知，1-男，2-女)',
    avatar varchar(512) DEFAULT NULL COMMENT '头像 URL',
    user_type tinyint(4) DEFAULT 1 COMMENT '用户类型 (1-系统用户，2-普通用户)',
    dept_id bigint(20) DEFAULT NULL COMMENT '所属部门 ID',
    status tinyint(4) DEFAULT 1 COMMENT '状态 (0-停用，1-正常)',
    login_ip varchar(128) DEFAULT NULL COMMENT '最后登录 IP',
    login_time datetime DEFAULT NULL COMMENT '最后登录时间',
    create_by varchar(64) DEFAULT NULL COMMENT '创建人',
    create_time datetime DEFAULT NULL COMMENT '创建时间',
    update_by varchar(64) DEFAULT NULL COMMENT '更新人',
    update_time datetime DEFAULT NULL COMMENT '更新时间',
    deleted tinyint(1) DEFAULT 0 COMMENT '逻辑删除标识',
    remark varchar(512) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id),
    UNIQUE KEY uk_username (username),
    KEY idx_dept_id (dept_id),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';

-- 角色表
CREATE TABLE IF NOT EXISTS sys_role (
    id bigint(20) NOT NULL AUTO_INCREMENT COMMENT '角色 ID',
    role_name varchar(64) NOT NULL COMMENT '角色名称',
    role_code varchar(64) NOT NULL COMMENT '角色编码',
    role_sort int(11) DEFAULT 0 COMMENT '排序',
    data_scope tinyint(4) DEFAULT 5 COMMENT '数据权限范围',
    status tinyint(4) DEFAULT 1 COMMENT '状态',
    create_by varchar(64) DEFAULT NULL COMMENT '创建人',
    create_time datetime DEFAULT NULL COMMENT '创建时间',
    update_by varchar(64) DEFAULT NULL COMMENT '更新人',
    update_time datetime DEFAULT NULL COMMENT '更新时间',
    deleted tinyint(1) DEFAULT 0 COMMENT '逻辑删除标识',
    remark varchar(512) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id),
    UNIQUE KEY uk_role_code (role_code),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统角色表';

-- 菜单表
CREATE TABLE IF NOT EXISTS sys_menu (
    id bigint(20) NOT NULL AUTO_INCREMENT COMMENT '菜单 ID',
    menu_name varchar(64) NOT NULL COMMENT '菜单名称',
    parent_id bigint(20) DEFAULT 0 COMMENT '父菜单 ID',
    menu_type tinyint(4) DEFAULT 1 COMMENT '菜单类型',
    path varchar(255) DEFAULT NULL COMMENT '路由地址',
    component varchar(255) DEFAULT NULL COMMENT '组件路径',
    permission varchar(64) DEFAULT NULL COMMENT '权限标识',
    icon varchar(64) DEFAULT NULL COMMENT '菜单图标',
    order_num int(11) DEFAULT 0 COMMENT '排序',
    status tinyint(4) DEFAULT 1 COMMENT '状态',
    visible tinyint(1) DEFAULT 1 COMMENT '是否可见',
    redirect varchar(255) DEFAULT NULL COMMENT '跳转链接',
    create_by varchar(64) DEFAULT NULL COMMENT '创建人',
    create_time datetime DEFAULT NULL COMMENT '创建时间',
    update_by varchar(64) DEFAULT NULL COMMENT '更新人',
    update_time datetime DEFAULT NULL COMMENT '更新时间',
    deleted tinyint(1) DEFAULT 0 COMMENT '逻辑删除标识',
    remark varchar(512) DEFAULT NULL COMMENT '备注',
    PRIMARY KEY (id),
    KEY idx_parent_id (parent_id),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统菜单表';

-- 用户角色关联表
CREATE TABLE IF NOT EXISTS sys_user_role (
    user_id bigint(20) NOT NULL COMMENT '用户 ID',
    role_id bigint(20) NOT NULL COMMENT '角色 ID',
    PRIMARY KEY (user_id, role_id),
    KEY idx_role_id (role_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户角色关联表';

-- 角色菜单关联表
CREATE TABLE IF NOT EXISTS sys_role_menu (
    role_id bigint(20) NOT NULL COMMENT '角色 ID',
    menu_id bigint(20) NOT NULL COMMENT '菜单 ID',
    PRIMARY KEY (role_id, menu_id),
    KEY idx_menu_id (menu_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='角色菜单关联表';
```

### 7.8 R__insert_dict_data.sql 示例（可重复执行）

```sql
-- 字典类型：用户性别
INSERT INTO sys_dict_type (dict_name, dict_type, status)
VALUES ('用户性别', 'user_gender', 1)
ON DUPLICATE KEY UPDATE dict_name = VALUES(dict_name);

-- 字典数据：用户性别
INSERT INTO sys_dict_data (dict_type_id, dict_label, dict_value, dict_sort, status)
VALUES
    ((SELECT id FROM sys_dict_type WHERE dict_type = 'user_gender'), '男', '1', 1, 1),
    ((SELECT id FROM sys_dict_type WHERE dict_type = 'user_gender'), '女', '2', 2, 1),
    ((SELECT id FROM sys_dict_type WHERE dict_type = 'user_gender'), '未知', '0', 0, 1)
ON DUPLICATE KEY UPDATE dict_label = VALUES(dict_label);
```

### 7.11 用户 - 角色关系

- 一个用户可以拥有多个角色
- 一个角色可以被多个用户拥有
- 通过 `sys_user_role` 表进行关联

### 7.12 角色 - 菜单关系

- 一个角色可以访问多个菜单
- 一个菜单可以被多个角色访问
- 通过 `sys_role_menu` 表进行关联

### 7.13 用户 - 部门关系

- 一个用户属于一个部门
- 一个部门可以包含多个用户
- 通过 `sys_user.dept_id` 字段关联

### 7.14 部门层级关系

- 部门支持多级树形结构
- 通过 `sys_dept.parent_id` 字段实现自关联
- 根部门的 `parent_id` 为 0

## 索引设计

### 7.15 索引命名规范

| 索引类型 | 命名前缀 |
|----------|----------|
| 主键索引 | PRIMARY KEY |
| 唯一索引 | uk_ |
| 普通索引 | idx_ |

### 7.16 索引设计原则

1. 主键必须存在且使用自增 ID
2. 外键字段必须建立索引
3. 查询频繁字段考虑建立索引
4. 区分度低的字段不适合单独建索引
5. 联合索引遵循最左前缀匹配原则

### 7.17 核心索引列表

| 表名 | 索引名 | 字段 | 类型 | 说明 |
|------|--------|------|------|------|
| sys_user | PRIMARY | id | 主键 | 主键索引 |
| sys_user | uk_username | username | 唯一索引 | 用户名唯一 |
| sys_user | idx_dept_id | dept_id | 普通索引 | 部门查询 |
| sys_user | idx_status | status | 普通索引 | 状态筛选 |
| sys_role | PRIMARY | id | 主键 | 主键索引 |
| sys_role | uk_role_code | role_code | 唯一索引 | 角色编码唯一 |
| sys_menu | PRIMARY | id | 主键 | 主键索引 |
| sys_menu | idx_parent_id | parent_id | 普通索引 | 子菜单查询 |
| sys_dept | PRIMARY | id | 主键 | 主键索引 |
| sys_dept | idx_parent_id | parent_id | 普通索引 | 子部门查询 |
