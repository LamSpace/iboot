# 18-国际化开发指南

## 1. 概述

本文档提供 iBoot 项目国际化开发的详细指南，包括如何添加翻译、使用翻译函数、以及最佳实践。

## 2. 添加翻译键

### 2.1 翻译文件命名规范

前端语言包采用模块化的 JSON 文件结构：

```
iboot-portal/src/locales/
├── zh-CN/                    # 简体中文
│   ├── common.json           # 通用词汇
│   ├── login.json            # 登录模块
│   ├── system/               # 系统管理模块
│   │   └── user.json
│   └── ...
└── en-US/                    # 英文
    └── （相同结构）
```

### 2.2 键名命名约定

使用 `.` 分隔的层级命名：

```
模块。功能。具体键
```

**示例**:
```
login.title                  # 登录页标题
login.username_placeholder   # 用户名输入框占位符
system.user.add_success      # 系统用户模块 - 添加成功提示
message.notice.popup.title   # 消息公告 - 弹窗标题
```

### 2.3 添加翻译的步骤

**步骤 1**: 确定翻译键的位置

根据功能模块选择对应的 JSON 文件：

| 模块 | 文件路径 |
|------|---------|
| 通用词汇 | `common.json` |
| 登录/注册 | `login.json`, `register.json` |
| 系统管理 | `system/user.json`, `system/role.json` 等 |
| 消息通知 | `message/message-center.json`, `message/notice.json` |
| 监控 | `monitor/online-user.json`, `monitor/server.json` 等 |

**步骤 2**: 在中文语言包中添加键

编辑 `iboot-portal/src/locales/zh-CN/xxx.json`:

```json
{
  "login": {
    "title": "用户登录",
    "username_placeholder": "请输入用户名",
    "password_placeholder": "请输入密码",
    "remember_me": "记住我",
    "forgot_password": "忘记密码？"
  }
}
```

**步骤 3**: 在英文语言包中添加对应翻译

编辑 `iboot-portal/src/locales/en-US/xxx.json`:

```json
{
  "login": {
    "title": "User Login",
    "username_placeholder": "Enter username",
    "password_placeholder": "Enter password",
    "remember_me": "Remember me",
    "forgot_password": "Forgot password?"
  }
}
```

**步骤 4**: 在组件中使用翻译

```vue
<template>
  <h2>{{ t('login.title') }}</h2>
  <el-input :placeholder="t('login.username_placeholder')" />
</template>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>
```

### 2.4 翻译检查清单

添加翻译前请确认：

- [ ] 翻译键遵循 `模块。功能。键` 命名规范
- [ ] 同时在 `zh-CN` 和 `en-US` 文件中添加
- [ ] 中英文翻译语义准确
- [ ] 没有硬编码中文文本
- [ ] 参数化翻译使用 `{0}`, `{name}` 等占位符

## 3. 前端组件使用翻译

### 3.1 基本用法

```typescript
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

// 在 script 中使用
const title = t('login.title')
</script>

<template>
  <!-- 文本内容 -->
  <div>{{ t('login.title') }}</div>

  <!-- 属性绑定 -->
  <el-input :placeholder="t('login.username_placeholder')" />

  <!-- 按钮文本 -->
  <el-button type="primary">
    {{ t('common.submit') }}
  </el-button>
</template>
```

### 3.2 参数化翻译

```json
// common.json
{
  "welcome": "欢迎，{0}！",
  "item_count": "共 {total} 项",
  "delete_confirm": "确定要删除 {name} 吗？"
}
```

```typescript
// 单个参数
t('welcome', '张三')
// 输出：欢迎，张三！

// 命名参数
t('item_count', { total: 100 })
// 输出：共 100 项

// 多个参数
t('delete_confirm', { name: '该用户' })
// 输出：确定要删除该用户吗？
```

### 3.3 在计算属性中使用

```typescript
const currentLabel = computed(() => {
  return locales.value.find(l => l.value === currentLocale.value)?.label
    || t('common.locale.zh-CN')

```

### 3.4 在函数中使用

```typescript

  ElMessage.warning(t('common.messages.not_permitted'))
```

### 3.5 时间格式化的语言处理

```typescript
const formatTime = (timeString: string) => {
  if (!timeString) return ''

  const date = new Date(timeString)
  const now = new Date()
  const diff = now.getTime() - date.getTime()

  // 使用 t() 获取当前语言的相对时间文本
  if (diff < 60 * 1000) {
    return t('message.messageCenter.notification.time.just_now')
  }

  if (diff < 60 * 60 * 1000) {
    const minutes = Math.floor(diff / (60 * 1000))
    return t('message.messageCenter.notification.time.minutes_ago')
      .replace('{0}', String(minutes))
  }

  // 使用动态 locale 进行日期格式化
  const locale = document.documentElement.lang || 'zh-CN'
  return date.toLocaleDateString(locale, {
    month: 'short',
    day: 'numeric',
    hour: '2-digit',
    minute: '2-digit'
  })
}
```

## 4. 后端使用翻译

### 4.1 使用 I18nUtil 工具类

```java
@Autowired
private I18nUtil i18nUtil;

// 获取基本消息
String message = i18nUtil.getMessage("user.login.success");
// 输出（中文）：登录成功
// 输出（英文）：Login successful

// 获取带参数的消息
String welcome = i18nUtil.getMessage("user.welcome", "张三");
// 输出（中文）：欢迎，张三！
// 输出（英文）：Welcome, Zhang San!
```

### 4.2 在 Service 中使用

```java
@Service
public class UserService {

    @Autowired
    private I18nUtil i18nUtil;

    public UserDTO login(String username, String password) {
        User user = userRepository.findByUsername(username);
        if (user == null) {
            throw new BusinessException("user.not_found");
        }
        if (!passwordEncoder.matches(password, user.getPassword())) {
            throw new BusinessException("user.password_error");
        }

        // 返回带消息码的成功响应
        return Result.success("user.login.success", convertToDTO(user));
    }
}
```

### 4.3 在 Controller 中使用

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public Result<UserDTO> create(@RequestBody UserCreateRequest request) {
        UserDTO user = userService.create(request);
        // 使用消息码返回成功响应
        return Result.success("user.create.success", user);
    }

    @DeleteMapping("/{id}")
    public Result<Void> delete(@PathVariable Long id) {
        userService.delete(id);
        return Result.success("user.delete.success", null);
    }
}
```

### 4.4 在异常处理中使用

```java
@Service
public class OrderService {

    @Autowired
    private I18nUtil i18nUtil;

    public void createOrder(OrderCreateRequest request) {
        if (request.getQuantity() > stockService.getStock(request.getProductId())) {
            // 抛出带消息码的业务异常
            throw new BusinessException("order.insufficient_stock");
        }
    }
}
```

### 4.5 全局异常自动处理

`GlobalExceptionHandler` 会自动处理异常的国际化：

```java
@ExceptionHandler(BusinessException.class)
public Result<Void> handleBusinessException(BusinessException e) {
    // 自动从 MessageSource 获取对应语言的消息
    String message = i18nUtil.getMessage(e.getMessageCode());
    return Result.error(message);
}
```

### 5.1 消息文件格式

## 6. 代码示例

### 6.1 完整登录页示例

```vue
<template>
  <div class="login-container">
    <h2>{{ t('login.title') }}</h2>

    <el-form :model="form" :rules="rules" ref="formRef">
      <el-form-item prop="username">
        <el-input
          v-model="form.username"
          :placeholder="t('login.username_placeholder')"
          prefix-icon="User"
        />
      </el-form-item>

      <el-form-item prop="password">
        <el-input
          v-model="form.password"
          type="password"
          :placeholder="t('login.password_placeholder')"
          prefix-icon="Lock"
          show-password
          @keyup.enter="handleLogin"
        />
      </el-form-item>

      <el-form-item>
        <el-checkbox v-model="form.remember">
          {{ t('login.remember_me') }}
        </el-checkbox>
      </el-form-item>

      <el-form-item>
        <el-button
          type="primary"
          :loading="loading"
          @click="handleLogin"
          style="width: 100%"
        >
          {{ t('login.submit') }}
        </el-button>
      </el-form-item>

      <div class="links">
        <router-link to="/forgot-password">
          {{ t('login.forgot_password') }}
        </router-link>
      </div>
    </el-form>
  </div>
</template>

<script setup lang="ts">
import { ref, reactive } from 'vue'
import { useI18n } from 'vue-i18n'
import { useRouter } from 'vue-router'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'

const { t } = useI18n()
const router = useRouter()
const userStore = useUserStore()

const formRef = ref()
const loading = ref(false)

const form = reactive({
  username: '',
  password: '',
  remember: false
})

const rules = {
  username: [
    { required: true, message: t('login.rules.username_required'), trigger: 'blur' }
  ],
  password: [
    { required: true, message: t('login.rules.password_required'), trigger: 'blur' }
  ]
}

const handleLogin = async () => {
  await formRef.value.validate(async (valid: boolean) => {
    if (!valid) return

    loading.value = true
    try {
      await userStore.login({
        username: form.username,
        password: form.password
      })
      ElMessage.success(t('login.messages.login_success'))
      router.push('/')
    } catch (error) {
      ElMessage.error(t('login.messages.login_failed'))
    } finally {
      loading.value = false
    }
  })
}
</script>
```

## 7. 检查清单

在提交代码前，请检查以下内容：

### 7.1 前端检查

- [ ] 所有用户可见文本都使用 `t()` 函数
- [ ] 没有在模板中硬编码中文
- [ ] 没有在 script 中硬编码中文（ElMessage 等）
- [ ] 占位符、按钮、标签、提示信息都已国际化
- [ ] 中英文语言包都有对应的翻译
- [ ] 翻译键命名遵循规范

### 7.2 后端检查

- [ ] Result 响应使用消息码而非硬编码文本
- [ ] 异常消息使用消息码
- [ ] 业务消息添加到资源束文件
- [ ] 中英文资源文件都有对应的翻译

### 7.3 常见遗漏点

| 位置 | 容易遗漏的文本 |
|------|---------------|
| Table 列头 | `:label="t('system.user.name')"` |
| 表单验证规则 | `message: t('login.rules.required')` |
| ElMessage | `ElMessage.success(t('common.success'))` |
| ElMessageBox | `title: t('common.confirm')` |
| Pagination | Element Plus 会自动翻译 |
| DatePicker | Element Plus 会自动翻译 |
| 时间格式化 | 需要使用 `document.documentElement.lang` |

## 8. 调试技巧

### 8.1 检查缺失翻译

如果翻译键不存在，vue-i18n 会回退到显示键名本身：

```typescript
// 如果 'common.nonexistent' 不存在
t('common.nonexistent')
// 输出：common.nonexistent
```

在浏览器控制台检查：

```javascript
// 检查当前语言
console.log(document.documentElement.lang)

// 检查已加载的语言包
console.log(window.$i18n?.messages)
```

### 8.2 强制刷新语言

在开发过程中切换语言测试：

```typescript
// 切换到英文
document.documentElement.lang = 'en-US'
localStorage.setItem('locale', 'en-US')
location.reload()
```
