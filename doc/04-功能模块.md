## 系统管理

### 4.1 功能概述

系统管理模块提供整个平台的基础管理能力，覆盖用户、角色、权限、菜单、数据字典、配置等核心功能，是 RBAC（Role-Based Access Control）权限模型的实现基础。

### 4.2 功能清单

| 功能 | 描述 | 导航路径 |
|------|------|----------|
| 用户管理 | 系统用户的增删改查、角色分配、状态管理 | 系统管理 > 用户管理 |
| 角色管理 | 角色的创建、编辑、删除、权限分配 | 系统管理 > 角色管理 |
| 权限管理 | 权限点的定义、分配与调整 | 系统管理 > 权限管理 |
| 菜单管理 | 系统菜单的配置、排序、图标设置 | 系统管理 > 菜单管理 |
| 数据字典 | 系统常量与枚举值的统一管理 | 系统管理 > 数据字典 |
| 配置管理 | 系统参数的动态配置 | 系统管理 > 配置管理 |

### 4.3 用户管理

**功能详情：**
- 用户信息的 CRUD 操作
- 用户与角色的关联管理
- 用户所属部门配置
- 用户状态管理（启用/停用）
- 用户登录日志查看

**数据表：**
- `sys_user` - 用户主表
- `sys_user_role` - 用户角色关联表
- `sys_dept` - 部门表

**后端实现：**

| 层级 | 类名 | 职责 |
|------|------|------|
| 接口层 | `UserController` | 处理用户相关的 HTTP 请求 |
| 应用层 | `UserApplicationService` | 编排用户业务流程 |
| 领域层 | `User` | 用户实体，包含领域行为 |
| 领域层 | `UserRepository` | 用户仓储接口 |
| 基础设施层 | `UserRepositoryImpl` | 用户仓储实现 |
| 基础设施层 | `UserMapper` | MyBatis Mapper 接口 |
| 基础设施层 | `UserPO` | 用户持久化对象 |

**典型流程：**
```
1. 管理员进入用户管理页面
2. 点击"新增用户"，填写用户名、密码、昵称、邮箱、手机号等信息
3. 选择所属部门和分配角色
4. 保存后，用户即可使用分配的账号登录
5. 后续可随时修改用户信息或调整角色
```

**DDD 流程说明：**
```
Controller (接口层)
    │
    ▼
UserApplicationService (应用层)
    │
    ├─► UserRepository (领域层 - 查询用户)
    ├─► PasswordService (领域层 - 加密密码)
    ├─► User.create() (领域层 - 创建实体)
    └─► UserRepository.save() (基础设施层 - 持久化)
```

```java
// 递归构建树形结构
public List<MenuTreeNode> buildTree(List<Menu> menus, Long parentId) {
    return menus.stream()
        .filter(m -> Objects.equals(m.getParentId(), parentId))
        .map(menu -> {
            MenuTreeNode node = convertToTreeNode(menu);
            node.setChildren(buildTree(menus, menu.getId()));
            return node;
        })
        .toList();
}
```

## 消息推送

### 4.12 功能概述

消息推送模块基于 SSE（Server-Sent Events）技术实现服务器向客户端的实时消息推送，支持站内消息通知、消息已读同步、系统通知等功能。

### 4.13 功能清单

| 功能 | 描述 | 导航路径 |
|------|------|----------|
| 站内消息 | 用户间的私信发送与接收 | 消息中心 > 我的消息 |
| 消息已读 | 消息已读状态同步 | 消息中心 > 我的消息 |
| 系统通知 | 系统公告和通知推送 | 消息中心 > 系统通知 |
| 实时推送 | 基于 SSE 的实时消息推送 | - |
| 在线状态 | 用户在线状态检测 | - |

### 4.14 技术架构

| 组件 | 描述 |
|------|------|
| SSE | Server-Sent Events，服务器推送协议 |
| Spring SseEmitter | Spring Framework SSE 支持 |
| Redis Pub/Sub | 集群模式下的消息广播 |
| CloudEvents | 事件格式规范 |

**推送模式：**
| 模式 | 描述 | 适用场景 |
|------|------|----------|
| 单播 | 发送给单个用户 | 私信通知 |
| 组播 | 发送给多个用户 | 群组通知 |
| 广播 | 发送给所有在线用户 | 系统公告 |

### 4.15 后端实现

| 层级 | 类名 | 职责 |
|------|------|------|
| 端点层 | `PushEndpoint` | SSE 连接入口，处理 `/api/push/connect` |
| 服务层 | `PushEventService` | 推送事件服务，支持单播/组播/广播 |
| 管理层 | `SseEmitterManager` | SSE 连接管理器，维护连接池 |
| 监听器 | `MessagePushListener` | 监听业务事件并触发推送 |
| 广播器 | `RedisPushEventBroadcaster` | Redis 集群事件广播 |
| 配置类 | `PushProperties` | 推送配置属性 |
| 事件类 | `PushEvent` | 推送事件数据模型（CloudEvents 兼容） |

**消息推送流程：**
```
1. 用户 A 发送消息给用户 B
   ↓
2. MessageApplicationService 保存消息
   ↓
3. 发布 MessageSentEvent 事件
   ↓
4. MessagePushListener 监听到事件（@TransactionalEventListener）
   ↓
5. PushEventService 构建 PushEvent
   ↓
6. SseEmitterManager 发送 SSE 消息给用户 B
   ↓
7. 用户 B 的浏览器接收并显示通知
```

**配置示例：**
```yaml
iboot:
  push:
    # 广播器类型：LOCAL（单机）或 REDIS（集群）
    broadcaster: local
    # SSE 配置
    sse:
      # 连接超时时间（毫秒）
      timeout: 3600000
      # 心跳间隔（秒）
      heartbeat-interval: 30
    # Redis 配置（集群模式使用）
    redis:
      channel: push:event
```

### 4.20 安全机制

- **认证方式**：JWT Token 通过 URL 参数传递（`?t=xxx`）
- **白名单**：`/api/push/**` 不在安全白名单中，必须认证
- **心跳保活**：30 秒心跳间隔，保持连接活跃
- **断线重连**：客户端自动重连机制（3 秒间隔）

```java
@PostMapping("/upload")
public Result<FileInfo> uploadFile(@RequestParam("file") MultipartFile file) {
    // 1. 校验文件类型和大小
    fileService.validateFile(file);

    // 2. 生成文件名
    String fileName = FileUtil.generateFileName(file.getOriginalFilename());

    // 4. 保存文件元数据到数据库
    FileInfo fileInfo = fileService.saveFileInfo(file, fileName);

    return Result.success(fileInfo);
}
```

**定时任务实现：**
```java
// 任务实体
@Data
@TableName("sys_job")
public class Job {
    private Long id;
    private String jobName;
    private String jobGroup;
    private String invokeTarget;  // 调用目标
    private String cronExpression;
    private String misfirePolicy;
    private Integer concurrent;
    private Integer status;
}

// 任务执行
@Component
public class JobExecution {

    @Autowired
    private Scheduler scheduler;

    // 添加定时任务
    public void addJob(Job job) {
        JobDetail jobDetail = JobBuilder.newJob(QuartzJobExecution.class)
            .withIdentity(job.getJobName(), job.getJobGroup())
            .build();

        CronTrigger trigger = TriggerBuilder.newTrigger()
            .withIdentity(job.getJobName(), job.getJobGroup())
            .withSchedule(CronScheduleBuilder.cronSchedule(job.getCronExpression()))
            .build();

        scheduler.scheduleJob(jobDetail, trigger);
    }
}
```

```java
@Service
public class ExcelExportService {

    public void exportToExcel(HttpServletResponse response) throws IOException {
        response.setContentType("application/vnd.ms-excel");
        response.setCharacterEncoding("utf-8");
        String fileName = URLEncoder.encode("用户数据报表", "UTF-8");
        response.setHeader("Content-Disposition",
            "attachment;filename=" + fileName + ".xlsx");

        try (Workbook workbook = new XSSFWorkbook()) {
            Sheet sheet = workbook.createSheet("用户数据");

            // 创建表头
            Row header = sheet.createRow(0);
            header.createCell(0).setCellValue("用户名");
            header.createCell(1).setCellValue("昵称");
            header.createCell(2).setCellValue("邮箱");

            // 填充数据
            List<User> users = userMapper.selectAll();
            int rowNum = 1;
            for (User user : users) {
                Row row = sheet.createRow(rowNum++);
                row.createCell(0).setCellValue(user.getUsername());
                row.createCell(1).setCellValue(user.getNickname());
                row.createCell(2).setCellValue(user.getEmail());
            }

            workbook.write(response.getOutputStream());
        }
    }
}
```
