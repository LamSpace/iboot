### 3.2 技术架构特点

1. **前后端分离**：前端负责展示与交互，后端提供 API 服务
2. **微服务就绪**：基于 Spring Boot 的模块化设计，可平滑演进为微服务
3. **容器化部署**：支持 Docker 容器化，便于水平扩展
4. **可观测性**：集成 Prometheus + Grafana + ELK 监控体系

## DDD 分层架构

### 3.3 领域驱动设计概述

iBoot 后端采用 DDD（Domain-Driven Design，领域驱动设计）架构模式，将系统划分为以下层次：

```
┌─────────────────────────────────────┐
│         Interfaces Layer            │  接口层
│    (Controllers, DTOs, Requests)    │
├─────────────────────────────────────┤
│       Application Layer             │  应用层
│  (Services, Application Events)     │
├─────────────────────────────────────┤
│          Domain Layer               │  领域层
│ (Entities, Value Objects, Events)   │
├─────────────────────────────────────┤
│      Infrastructure Layer           │  基础设施层
│  (Repositories, Persistence,外部服务) │
└─────────────────────────────────────┘
```

**层间依赖规则：**

```
┌─────────────────────────────────────────────────────────┐
│                    依赖倒置原则 (DIP)                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  接口层 ──────────────────────► 应用层                  │
│     │                              │                    │
│     │                              │                    │
│     ▼                              ▼                    │
│  基础设施层 ◄─────────────────── 领域层                 │
│     │              (核心业务，不依赖外部)                │
│     │                              │                    │
│     └──────────┬───────────────────┘                    │
│                │                                        │
│                ▼                                        │
│         仓储接口实现、持久化实现、外部服务实现              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**核心设计原则：**

| 原则 | 描述 | 实现方式 |
|------|------|----------|
| 单一职责 | 每层只负责特定职责 | 严格的包结构划分 |
| 依赖倒置 | 高层不依赖低层，低层实现高层接口 | 仓储模式、面向接口编程 |
| 领域纯净 | 领域层不依赖外部框架和技术细节 | 纯 POJO 实现、接口隔离 |
| 边界清晰 | 层与层之间通过 DTO 传递数据 | 明确的对象转换边界 |

### 3.4 各层职责

#### 接口层（Interfaces Layer）

**职责：**
- 接收用户请求，进行参数校验
- 调用应用服务处理业务
- 返回响应结果
- 处理 HTTP 相关的跨域、认证等

**组件：**
- `@RestController`：RESTful API 控制器
- `@RequestBody` / `@ResponseBody`：请求/响应体
- DTO（Data Transfer Object）：数据传输对象
- Request/Response：请求/响应对象

**代码示例 - UserController：**

```java
package com.iboot.admin.interfaces.controller;

import com.iboot.admin.interfaces.dto.user.CreateUserRequest;
import com.iboot.admin.interfaces.dto.user.UserResponse;
import com.iboot.admin.application.service.UserApplicationService;
import com.iboot.admin.application.dto.UserDTO;
import io.swagger.v3.oas.annotations.tags.Tag;
import io.swagger.v3.oas.annotations.Operation;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

/**
 * 用户管理控制器
 *
 * 处理用户相关的 HTTP 请求，包括用户创建、查询、更新、删除等操作
 *
 * @author iBoot
 */
@Tag(name = "用户管理")
@RestController
@RequestMapping("/api/system/users")
@RequiredArgsConstructor
public class UserController {

    private final UserApplicationService userApplicationService;

    @PostMapping
    @Operation(summary = "创建用户")
    public ResponseEntity<UserResponse> createUser(
            @Valid @RequestBody CreateUserRequest request) {
        UserDTO userDTO = userApplicationService.createUser(request.toCommand());
        return ResponseEntity.ok(UserResponse.fromDTO(userDTO));
    }

    @GetMapping("/{id}")
    @Operation(summary = "获取用户详情")
    public ResponseEntity<UserResponse> getUser(@PathVariable Long id) {
        UserDTO userDTO = userApplicationService.getUserById(id);
        return ResponseEntity.ok(UserResponse.fromDTO(userDTO));
    }

    @GetMapping
    @Operation(summary = "分页查询用户")
    public ResponseEntity<PageResult<UserResponse>> getUsers(
            @RequestParam(defaultValue = "1") int pageNum,
            @RequestParam(defaultValue = "10") int pageSize) {
        PageResult<UserDTO> page = userApplicationService.getUsers(pageNum, pageSize);
        return ResponseEntity.ok(page.map(UserResponse::fromDTO));
    }

    @PutMapping("/{id}")
    @Operation(summary = "更新用户")
    public ResponseEntity<UserResponse> updateUser(
            @PathVariable Long id,
            @Valid @RequestBody CreateUserRequest request) {
        UserDTO userDTO = userApplicationService.updateUser(id, request.toCommand());
        return ResponseEntity.ok(UserResponse.fromDTO(userDTO));
    }

    @DeleteMapping("/{id}")
    @Operation(summary = "删除用户")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        userApplicationService.deleteUser(id);
        return ResponseEntity.noContent().build();
    }
}
```

#### 应用层（Application Layer）

**职责：**
- 编排业务流程
- 调用领域服务和仓储
- 处理事务
- 发布领域事件

**组件：**
- `@Service`：应用服务
- Application DTO：应用层数据传输对象
- Application Events：应用事件

**代码示例 - UserApplicationService：**

```java
package com.iboot.admin.application.service;

import com.iboot.admin.application.dto.UserDTO;
import com.iboot.admin.application.dto.command.CreateUserCommand;
import com.iboot.admin.application.dto.command.UpdateUserCommand;
import com.iboot.admin.domain.model.entity.User;
import com.iboot.admin.domain.model.event.UserCreatedEvent;
import com.iboot.admin.domain.model.event.UserUpdatedEvent;
import com.iboot.admin.domain.repository.UserRepository;
import com.iboot.admin.domain.service.PasswordService;
import com.iboot.admin.infrastructure.persistence.po.UserPO;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.context.ApplicationEventPublisher;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

/**
 * 用户应用服务
 *
 * 负责用户相关的业务逻辑编排和流程处理
 *
 * @author iBoot
 */
@Slf4j
@Service
@RequiredArgsConstructor
@Transactional(rollbackFor = Exception.class)
public class UserApplicationService {

    private final UserRepository userRepository;
    private final PasswordService passwordService;
    private final ApplicationEventPublisher eventPublisher;

    /**
     * 创建用户
     *
     * 业务逻辑：
     * 1. 校验用户名是否存在
     * 2. 加密密码
     * 3. 创建用户实体
     * 4. 保存用户
     * 5. 发布用户创建事件
     *
     * @param command 创建用户命令
     * @return 用户 DTO
     */
    public UserDTO createUser(CreateUserCommand command) {
        // 1. 校验用户名是否存在
        if (userRepository.existsByUsername(command.username())) {
            throw new BusinessException("用户名已存在");
        }

        // 2. 加密密码
        String encodedPassword = passwordService.encode(command.password());

        // 3. 创建用户实体
        User user = User.create(
            command.username(),
            encodedPassword,
            command.nickname(),
            command.email(),
            command.phone(),
            command.deptId(),
            command.roleIds()
        );

        // 4. 保存用户
        userRepository.save(user);

        // 5. 发布领域事件
        eventPublisher.publishEvent(new UserCreatedEvent(this, user.getId()));

        log.info("用户创建成功，userId: {}", user.getId());

        return UserDTO.fromDomain(user);
    }

    /**
     * 更新用户
     *
     * @param id 用户 ID
     * @param command 更新用户命令
     * @return 用户 DTO
     */
    public UserDTO updateUser(Long id, UpdateUserCommand command) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new BusinessException("用户不存在"));

        // 更新用户信息
        user.updateInfo(command.nickname(), command.email(), command.phone(), command.deptId());

        // 更新角色
        if (command.roleIds() != null) {
            user.updateRoles(command.roleIds());
        }

        userRepository.save(user);

        // 发布领域事件
        eventPublisher.publishEvent(new UserUpdatedEvent(this, user.getId()));

        return UserDTO.fromDomain(user);
    }

    /**
     * 删除用户
     *
     * @param id 用户 ID
     */
    public void deleteUser(Long id) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new BusinessException("用户不存在"));

        // 逻辑删除
        user.delete();
        userRepository.save(user);

        log.info("用户删除成功，userId: {}", id);
    }

    /**
     * 修改密码
     *
     * @param id 用户 ID
     * @param oldPassword 旧密码
     * @param newPassword 新密码
     */
    public void changePassword(Long id, String oldPassword, String newPassword) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new BusinessException("用户不存在"));

        // 验证旧密码
        if (!passwordService.matches(oldPassword, user.getPassword())) {
            throw new BusinessException("原密码不正确");
        }

        // 更新密码
        user.changePassword(passwordService.encode(newPassword));
        userRepository.save(user);

        log.info("密码修改成功，userId: {}", id);
    }

    /**
     * 根据 ID 获取用户
     *
     * @param id 用户 ID
     * @return 用户 DTO
     */
    @Transactional(readOnly = true)
    public UserDTO getUserById(Long id) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new BusinessException("用户不存在"));
        return UserDTO.fromDomain(user);
    }

    /**
     * 分页查询用户
     *
     * @param pageNum 页码
     * @param pageSize 每页大小
     * @return 分页结果
     */
    @Transactional(readOnly = true)
    public PageResult<UserDTO> getUsers(int pageNum, int pageSize) {
        Page<User> page = userRepository.findAll(pageNum, pageSize);
        return PageResult.of(
            page.getContent().stream()
                .map(UserDTO::fromDomain)
                .toList(),
            page.getTotalElements(),
            pageNum,
            pageSize
        );
    }
}
```

#### 领域层（Domain Layer）

**职责：**
- 实现核心业务逻辑
- 定义领域模型
- 定义仓储接口
- 发布领域事件

**组件：**
- Entity：实体类
- Value Object：值对象
- Repository Interface：仓储接口
- Domain Service：领域服务
- Domain Event：领域事件

**代码示例 - User 实体：**

```java
package com.iboot.admin.domain.model.entity;

import com.iboot.admin.domain.model.vo.UserProfile;
import com.iboot.admin.domain.model.valueobject.RoleId;
import com.iboot.admin.common.exception.BusinessException;
import lombok.Getter;
import lombok.extern.slf4j.Slf4j;

import java.time.LocalDateTime;
import java.util.ArrayList;
import java.util.List;
import java.util.Set;
import java.util.stream.Collectors;

/**
 * 用户实体类
 *
 * 用户是系统的核心领域模型，包含用户的基本信息、角色、权限等
 *
 * @author iBoot
 */
@Slf4j
@Getter
public class User {

    private Long id;
    private UserProfile profile;
    private String password;
    private Long deptId;
    private List<RoleId> roleIds;
    private Integer status;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private boolean deleted;

    // 状态常量
    public static final int STATUS_DISABLED = 0;
    public static final int STATUS_NORMAL = 1;

    /**
     * 创建用户
     *
     * @param username 用户名
     * @param password 加密后的密码
     * @param nickname 昵称
     * @param email 邮箱
     * @param phone 手机号
     * @param deptId 部门 ID
     * @param roleIds 角色 ID 列表
     * @return 用户实体
     */
    public static User create(String username, String password, String nickname,
                               String email, String phone, Long deptId, List<Long> roleIds) {
        User user = new User();
        user.profile = new UserProfile(username, nickname, email, phone);
        user.password = password;
        user.deptId = deptId;
        user.roleIds = roleIds != null ? roleIds.stream().map(RoleId::new).toList() : new ArrayList<>();
        user.status = STATUS_NORMAL;
        user.createTime = LocalDateTime.now();
        user.updateTime = LocalDateTime.now();
        user.deleted = false;
        return user;
    }

    /**
     * 更新用户信息
     *
     * @param nickname 昵称
     * @param email 邮箱
     * @param phone 手机号
     * @param deptId 部门 ID
     */
    public void updateInfo(String nickname, String email, String phone, Long deptId) {
        this.profile = this.profile.update(nickname, email, phone);
        this.deptId = deptId;
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 更新角色
     *
     * @param roleIds 角色 ID 列表
     */
    public void updateRoles(List<Long> roleIds) {
        this.roleIds = roleIds.stream().map(RoleId::new).toList();
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 修改密码
     *
     * @param newPassword 新密码
     */
    public void changePassword(String newPassword) {
        this.password = newPassword;
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 删除用户（逻辑删除）
     */
    public void delete() {
        this.deleted = true;
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 启用用户
     */
    public void enable() {
        this.status = STATUS_NORMAL;
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 禁用用户
     */
    public void disable() {
        this.status = STATUS_DISABLED;
        this.updateTime = LocalDateTime.now();
    }

    /**
     * 获取角色 ID 集合
     */
    public Set<Long> getRoleIds() {
        return roleIds.stream().map(RoleId::getId).collect(Collectors.toSet());
    }

    /**
     * 检查用户是否有效
     */
    public void checkStatus() {
        if (deleted) {
            throw new BusinessException("用户已删除");
        }
        if (status != STATUS_NORMAL) {
            throw new BusinessException("用户已禁用");
        }
    }
}
```

```java
package com.iboot.admin.domain.model.vo;

import lombok.EqualsAndHashCode;
import lombok.Getter;

/**
 * 用户档案值对象
 *
 * 值对象是不可变的，通过属性值来定义
 * 用于封装用户的基本信息
 *
 * @author iBoot
 */
@Getter
@EqualsAndHashCode
public class UserProfile {

    private final String username;
    private final String nickname;
    private final String email;
    private final String phone;

        this.username = username;
        this.nickname = nickname;
        this.email = email;
        this.phone = phone;
    }

    /**
     * 更新信息（返回新的值对象）
     */
    public UserProfile update(String nickname, String email, String phone) {
        return new UserProfile(
            this.username,
            nickname != null ? nickname : this.nickname,
            email != null ? email : this.email,
            phone != null ? phone : this.phone
        );
```

**代码示例 - UserRepositoryImpl 仓储实现：**

```java
package com.iboot.admin.infrastructure.repository;

import com.iboot.admin.domain.model.entity.User;
import com.iboot.admin.domain.model.vo.UserProfile;
import com.iboot.admin.domain.repository.UserRepository;
import com.iboot.admin.infrastructure.persistence.mapper.UserMapper;
import com.iboot.admin.infrastructure.persistence.po.UserPO;
import com.github.pagehelper.PageHelper;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Repository;

import java.util.List;
import java.util.Optional;

/**
 * 用户仓储实现类
 *
 * 实现 UserRepository 接口，负责用户实体的数据持久化操作
 *
 * @author iBoot
 */
@Repository
@RequiredArgsConstructor
public class UserRepositoryImpl implements UserRepository {

    private final UserMapper userMapper;

    @Override
    public Optional<User> findById(Long id) {
        UserPO po = userMapper.selectById(id);
        return Optional.ofNullable(convertToDomain(po));
    }

    @Override
    public Optional<User> findByUsername(String username) {
        UserPO po = userMapper.selectByUsername(username);
        return Optional.ofNullable(convertToDomain(po));
    }

    @Override
    public boolean existsByUsername(String username) {
        return userMapper.countByUsername(username) > 0;
    }

    @Override
    public void save(User user) {
        if (user.getId() == null) {
            // 新增
            UserPO po = convertToPO(user);
            userMapper.insert(po);
            // 回填 ID
            user = reflectId(po.getId());
        } else {
            // 更新
            userMapper.update(convertToPO(user));
        }
    }

    @Override
    public void deleteById(Long id) {
        userMapper.deleteById(id);
    }

    @Override
    public org.springframework.data.domain.Page<User> findAll(int pageNum, int pageSize) {
        PageHelper.startPage(pageNum, pageSize);
        List<UserPO> pos = userMapper.selectAll();
        com.github.pagehelper.Page<UserPO> page = (com.github.pagehelper.Page<UserPO>) pos;

        List<User> users = page.getResult().stream()
            .map(this::convertToDomain)
            .toList();

        return new org.springframework.data.domain.PageImpl<>(
            users,
            org.springframework.data.domain.PageRequest.of(pageNum - 1, pageSize),
            page.getTotal()
        );
    }

    @Override
    public List<User> findByRoleId(Long roleId) {
        List<UserPO> pos = userMapper.selectByRoleId(roleId);
        return pos.stream().map(this::convertToDomain).toList();
    }

    /**
     * PO 转 Domain
     */
    private User convertToDomain(UserPO po) {
        if (po == null) {
            return null;
        }
        User user = new User();
        user.setId(po.getId());
        user.setProfile(new UserProfile(po.getUsername(), po.getNickname(), po.getEmail(), po.getPhone()));
        user.setPassword(po.getPassword());
        user.setDeptId(po.getDeptId());
        user.setStatus(po.getStatus());
        user.setCreateTime(po.getCreateTime());
        user.setUpdateTime(po.getUpdateTime());
        user.setDeleted(po.getDeleted());
        return user;
    }

    /**
     * Domain 转 PO
     */
    private UserPO convertToPO(User user) {
        if (user == null) {
            return null;
        }
        UserPO po = new UserPO();
        po.setId(user.getId());
        po.setUsername(user.getProfile().getUsername());
        po.setNickname(user.getProfile().getNickname());
        po.setEmail(user.getProfile().getEmail());
        po.setPhone(user.getProfile().getPhone());
        po.setPassword(user.getPassword());
        po.setDeptId(user.getDeptId());
        po.setStatus(user.getStatus());
        po.setDeleted(user.isDeleted());
        return po;
    }
}
```

## 缓存架构

### 3.15 二级缓存架构

```
┌───────────────────────────────────────────┐
│            应用层                          │
│  ┌─────────────────────────────────────┐ │
│  │         TwoLevelCache               │ │
│  │  ┌─────────────┐   ┌─────────────┐  │ │
│  │  │  L1 Cache   │   │  L2 Cache   │  │ │
│  │  │  (Caffeine) │◄─►│   (Redis)   │  │ │
│  │  └─────────────┘   └─────────────┘  │ │
│  └─────────────────────────────────────┘ │
└───────────────────────────────────────────┘
              │                   │
              │                   │ Pub/Sub
              │                   │ (失效通知)
              ▼                   ▼
┌───────────────────────────────────────────┐
│              数据源                        │
│         MySQL / External API              │
└───────────────────────────────────────────┘
```

### 3.16 缓存策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| Cache-Aside | 先读缓存，未命中则读 DB 并回填 | 读多写少 |
| Write-Through | 写缓存同时写 DB | 强一致性 |
| Write-Behind | 先写缓存，异步写 DB | 高并发写 |
| 失效策略 | TTL、LRU、LFU | 内存控制 |
