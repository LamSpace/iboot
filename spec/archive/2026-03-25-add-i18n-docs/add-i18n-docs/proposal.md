# Proposal: Add Internationalization Documentation

## Why

iBoot 项目已完成全面的国际化（i18n）改造，包括：
- 前端 vue-i18n v9+ 完整配置与多语言包结构
- 后端 Spring MessageSource 配置与消息资源束
- 语言切换 API 与 UI 组件
- 完整的规格说明文档（spec/specs/internationalization/spec.md）

然而，现有 `doc/` 目录下的文档缺少国际化相关内容，导致：
1. 开发人员无法从文档了解 i18n 架构与使用方式
2. 新增翻译时缺少规范的流程指导
3. 多语言支持的设计理念未能在团队内传承

本文档提案旨在完善 `doc/` 目录，补充国际化相关文档。

## What Changes

### 新增文档

1. **`doc/17-国际化设计.md`** - 国际化架构设计文档
   - 前端 i18n 架构（vue-i18n 配置、语言包结构、加载机制）
   - 后端 i18n 架构（MessageSource 配置、资源束结构、LocaleResolver）
   - 语言切换流程（前后端交互、Cookie/LocalStorage 持久化）
   - 最佳实践与常见问题

2. **`doc/18-国际化开发指南.md`** - 开发者使用指南
   - 如何添加新的翻译键
   - 前端组件中使用 t() 函数
   - 后端使用 I18nUtil 获取消息
   - 翻译文件命名规范
   - 代码示例与检查清单

### 更新文档

3. **`doc/README.md`** - 更新文档导航
   - 在"后端开发人员"列表中添加国际化文档链接
   - 在"技术栈概览"中突出 vue-i18n 和 MessageSource

4. **`doc/02-技术栈.md`** - 更新技术栈说明
   - 前端技术栈添加 vue-i18n v9+
   - 后端技术栈添加 Spring MessageSource

5. **`doc/06-API 集成.md`** - 更新 API 文档
   - 添加语言切换 API（`/api/i18n/locale`）说明
   - 添加消息码响应格式说明

## Impact

### 受影响的规范
- 无直接影响，本文档仅补充说明既有实现

### 受影响的代码
- 无代码变更，仅文档更新

### API 影响
- 无 API 变更

### 用户影响
- 无直接影响，终端用户无感知

### 开发人员影响
- 新增开发人员在查阅文档时可快速了解 i18n 架构
- 提供标准化的翻译添加流程，减少错误

### 风险评估
- **风险等级**：低
- **原因**：仅文档变更，不涉及代码修改
- **缓解措施**：确保文档内容与实际实现一致
